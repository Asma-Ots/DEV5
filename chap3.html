Le Modele OSI

1) Les Composants matériel d’un Réseau informatique :

Les cartes réseau : également appelées cartes d'interface réseau (NIC), sont des composants matériels qui permettent à un ordinateur de se connecter à un réseau, tel qu'Internet ou un réseau local. Elles sont installées à l'intérieur de l'ordinateur et sont généralement connectées à une prise Ethernet ou Wi-Fi. SWITCH : Un switch (ou commutateur en français) est un dispositif réseau qui permet de connecter plusieurs périphériques sur un réseau local (LAN). Le switch est utilisé pour acheminer le trafic réseau entre les périphériques connectés en utilisant leur adresse MAC, ce qui permet une communication efficace et rapide entre eux. Les switches peuvent être de différentes tailles, avec un nombre de ports variable pour s'adapter aux besoins du réseau. Ils peuvent également être gérés ou non gérés, selon la complexité du réseau à gérer. Les switches gérés offrent des fonctionnalités supplémentaires telles que la surveillance du trafic réseau et la sécurité renforcée. Hub : est un dispositif de réseau permettant de connecter plusieurs périphériques sur un même réseau local (LAN). Contrairement aux switches, les hubs transmettent les données reçues à tous les périphériques connectés, ce qui peut entraîner une saturation du réseau en cas de trafic important. Les hubs sont généralement utilisés dans les réseaux de petite taille car ils ne disposent pas des fonctionnalités de gestion et de contrôle du trafic des switches. Les hubs peuvent être de différentes tailles, avec un nombre de ports variable pour s'adapter aux besoins du réseau. Cependant, en raison de leur faible coût, les hubs sont devenus largement obsolètes et ont été remplacés par des switches plus performants.

Le Modèle OSI :

est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques liées à la communication de données entre les appareils de réseau. Le modèle OSI est souvent utilisé comme référence pour la conception et la mise en œuvre de réseaux informatiques. Les sept couches du modèle OSI sont les suivantes :

1) La couche physique

2) La couche liaison de données

3) La couche réseau

4) La couche transport

5) La couche session

6) La couche présentation

7) La couche application

3) Le Modèle TCP/IP

Le modèle TCP/IP (Transmission Control Protocol/Internet Protocol) est un modèle de référence pour les communications de réseau. Il est largement utilisé dans la conception et la mise en œuvre des réseaux informatiques modernes. Contrairement au modèle OSI, qui divise les communications en sept couches distinctes, le modèle TCP/IP divise les communications en quatre couches principales. Ce modèle comprend la couche d'application, la couche transport, la couche réseau et la couche liaison. Le modèle TCP/IP est utilisé pour assurer une communication efficace entre les différents périphériques et réseaux informatiques. Chaque couche est responsable d'une partie spécifique du processus de communication et permet une communication efficace et sécurisée sur le réseau.

4) L’adressage :

Les communications dans un réseau obéissent à des règles Adressage , architecture et les Protocoles . un adressage IP : est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient. Il y’a deux mode d’adressage IPV4 sur (32bites) et IPV6 sur (128bites) .

5) L’adresse IPV4 :

L'adresse IPv4 est une adresse de protocole de communication utilisée pour identifier les périphériques sur un réseau. Elle est basée sur le protocole Internet Protocol version 4 (IPv4), qui est le plus utilisé à ce jour. Les adresses IPv4 sont des adresses de 32 bits, représentées sous forme de quatre octets séparés par des points, comme par exemple 192.168.0.1. Il existe plusieurs classes d'adresses IPv4, chacune avec une plage de valeurs d'adresses possibles. Les adresses de classe A, par exemple, sont utilisées pour les réseaux de grande taille, tandis que les adresses de classe C sont utilisées pour les réseaux de petite taille. Les adresses IPv4 peuvent être attribuées de manière statique ou dynamique à l'aide de protocoles tels que le DHCP. Cependant, l'utilisation croissante d'appareils connectés a entraîné une pénurie d'adresses IPv4 disponibles. Cela a conduit au développement d'IPv6, un protocole de communication plus récent qui utilise des adresses de 128 bits pour fournir un plus grand nombre d'adresses uniques et résoudre le problème de la pénurie d'adresses IPv4.
